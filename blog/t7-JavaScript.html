Ruby vs. JavaScript

Hashes? Objects?

2/27/2015

Until now, we have been discussing Ruby only. But today, I want us to talk about a new programming language called JavaScript. JavaScript is a great language for the web browser, with which we can take control of what we want the brower to do. For the sake of our discussion, we will tune our focus on Ruby hashes and JavaScript objects. Hopefully by the end of reading this entry, you will get an idea of some of the similarities and differences of the two. Let's start off by looking at the how the Ruby hashes and JavaScript objects look like.

Now that we've determined what Ruby hash and JavaScript object look like, here is a list of some similarities:
  • They are enclosed by a set of curly brackets.
  • They are given a variable name.
  • Each element within the brackets has two parts. For Ruby, they are called key and value; for JavaScript, property and value.
  • Values can be accessed using their corresponding key (Ruby) or property (JavaScript).

How about some differences between a Ruby hash and a JavaScript object? Here is a list:

  • They use different syntaxes for their elements. Hashes use ("=>") while objects use (":")
  • When accessing values in the set, hashes require the key to go inside square brackets ("[]"). On the other hand, values of the JavaScript objects are accessed by calling the propery name as a method onto the object.
